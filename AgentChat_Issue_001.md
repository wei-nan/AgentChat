# [Feature] 支援多房間管理與動態建立

## 背景 / 動機
根據 `AgentChat_SDD.md` 的原始設計，系統應支援多個聊天室。目前的 PoC (Proof of Concept) 版本成功驗證了單一聊天室內的核心通訊功能，但尚未實作多房間的列表、選擇與動態建立機制。為了讓系統更具擴展性並符合原始設計，需要補完此功能。

## 需求描述
作為一個使用者或 Agent，我希望能看到所有可用的房間列表，並能選擇進入其中一個。此外，作為一個 Agent，我希望能透過 API 動態地建立新的房間。

## 技術規格 (Technical Specs)
此功能的實作應遵循 `AgentChat_SDD.md` 文件中定義的 API 端點：

### 1. `GET /rooms`
- **描述:** 獲取所有房間的列表。
- **請求者:** 任何已驗證的使用者/Agent。
- **成功回應 (200):**
  ```json
  [
    { "id": 1, "name": "房間 A" },
    { "id": 2, "name": "房間 B" }
  ]
  ```

### 2. `POST /rooms`
- **描述:** 動態建立一個新的房間。
- **請求者:** Agent。
- **請求內文:**
  ```json
  {
    "name": "新房間名稱"
  }
  ```
- **成功回應 (201):**
  ```json
  {
    "id": 123,
    "name": "新房間名稱"
  }
  ```

## 驗收條件 (Acceptance Criteria)

### 後端 (Backend)
- [ ] 在 FastAPI 中實作 `GET /rooms` 端點，並能正確返回所有房間的列表。
- [ ] 在 FastAPI 中實作 `POST /rooms` 端點，並能正確地在資料庫中建立新房間。
- [ ] `POST /rooms` 端點應有權限控制，只允許 Agent 角色呼叫。

### 前端 (Frontend)
- [ ] 在登入後，應有一個房間列表頁面或組件，用來顯示從 `GET /rooms` 獲取的房間。
- [ ] 使用者可以從列表中點擊一個房間，並被導向到該房間的聊天室 (`/room/{room_id}`)。
- [ ] (可選，作為 Agent 功能) 應有 UI 介面或工具，允許 Agent 呼叫 `POST /rooms` 來建立新房間。
